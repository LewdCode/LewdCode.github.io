ReadMe:

This application was written in Java and designed to be platform independent. It is recommended that you create a folder somewhere called StreamTweet and run the jar file from there as the application will generate files in the folder it is run from. To use, you have to go through a few steps, but these will, in the end, save you a lot of time. 1) You must have a Twitter Application registered to the account you wish to Tweet from. To create an application go to https://dev.twitter.com/apps and follow the steps there. 2) You have to make sure that when entering the keys into the text boxes, there are no spaces. Blank Spaces will royally mess up the connection to the Twitter4j API which is used to send the Tweet. 3) You must enter a value in the Delay box. Trying to start Sending without a delay will cause the application to crash. ----------------- Once you have a Twitter Application created, open the StreamTweet application and enter your appropriate details. The Keys are only handled by the Twitter4j API which connects to Twitter. This method is used because the developer of StreamTweet had a brain fart trying to learn how to use oAuth without callback servers. If you would like to save the keys you've entered for future use, click the Save Keys to Disk. When you next open the application, you will be prompted to load keys if they are found on the system. A small bug has been found that you have to click your choice twice. It is not recommended that you save keys to disk if you use a shared computer. After entering your keys, you can enter your desired message into the Tweet to Send field. This can include links but is limited to 135 readable characters. If you have the StreamTweet Plus, you can opt for SmartTweets, which grab data from the Hitbox API, and allow a media link to be provided. SmartTweets are not editable, but Standard Tweets are.
